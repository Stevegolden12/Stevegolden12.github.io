If you have worked with Vanilla JavaScript before you know about selecting, traversing, and manipulating the DOM. Selecting is when you target a certain point in the DOM. Traversing is moving from point to point within the DOM. Manipulation is when you actually alter something within the DOM. Usually you do that in that order which makes sense. You cannot move or alter something if you have no place to start from.

We are going to take that with the D3 library. First thing you want to do is “select” a point of the DOM you want to either transverse or manipulate from. We usually do this by starting off using “d3.” then using the select() method. Also a big thing in JavaScript is chaining methods together and we are going to start chaining to the select() method. The append() method will help us attached (manipulate) a DOM element to another DOM element. Lastly we are also going to alter (manipulate) the DOM element we just append by using the text() method. Below is an example of how this works:

d3.select(“ul”).append(“li”).text(“list item 1”)

Prefer to write this as the above and means the same thing:

  d3.select(“ul”) //Selecting the first ul tag
  .append(“li”) //Add a li tag right after the ul tag
  .text(“list item 1”); /*Change the text of the li tag of "list item 1" */

Accessibility guidelines for UX Designers: Device independent design

by Steve Golden

1/3/19 at 11 p.m.

Going to continue to discuss the article “Accessibility guidelines for UX Designers’ by Avinash Kaur. The second part of her guideline is device independent design. Just give the user more than one way to look at the content because no one way can be accessible to all people and some ways hinder accessibility. Avinash Kaur looks at 5 help with being device independent design:

  • Interaction
  • Sensory characteristics
  • Alternative to device motion
  • Complex pointer gestures
  • Orientation of content

For Interactions you must make sure that the user knows there is new content if new content does show up and how to get out of the interaction.. In the W3C they mentioned custom tooltips and sub-menus that pop up and users as these types of interactions. They suggest that if you do use these make sure the user knows what is going on.

In the article they say, “Don’t provide instructions which rely solely on sensory characteristics of components such as shape, colour, size, visual location, orientation or sound”. An example of this is a pomodoro clock. When the clock switches from work to break and break to work time there is a beep. Those that are hearing impaired might have a challenge with this so a suggestion is to have a label above the time showing which side the timer is on.

When working you always want to have alternatives to device motion and complex pointer gestures. So if there is a feature on your phone that if you tilt it will affect the app or site this will be a concern. HOWEVER in the W3C they recognize that some of these effects are needed to use these features which is are the exceptions to this rule.

Lastly if you are working on the web make sure that your application/site can be viewed on either portrait or landscape. This is called orientation of content and examples of this are people with vision problems wanting to see family pictures on their phone on landscape rather than portrait. Or maybe instead of a family picture it is a bank check. One of the examples in W3C mentions is a piano application that you want to use on landscape rather than portrait.

Passing Data from Child to Parent Components

by Steve Golden

12/14/18 at 8 a.m.

One of React’s great strengths is also one of its greatest weaknesses. Components are a wonderful way to code because it helps to separate code to make it easier to read and reuse. The challenge with components is that while you are passing data it can get difficult to move it around components because every time data goes through a component you must make sure it passes through it.

The easiest way to do this is passing data Parent Component to Child Component. This is where the Child component is used in the Parent Component. There are times where you will need to pass data from the Child component to Parent Component. This takes a little more work and understanding on how to do it.

The way if your going to work with React without any other libraries to pass data from Child to Parent Components is using functions and callback functions.

  • First step is to make a function in the Parent Component where you are going to setState and having a parameter of the value you are going to change.
  • Second Step is to pass the function to the Child component.
  • Third Step is to have a callback function which will be a function that is using the function passed from the Parent Component. Yet this callback function will be used in the Child Component.

The following is an example of data passing through Child to Parent Component:

class ParentComponent extends React.Component{
  constructor(props){
    super(props)
      this.state = {
        input: ‘’,
      }
    this.handleChange = this.handleChange.bind(this);
    }

handleChange(newVal){
  this.setState({
     input: newVal,
  })
}

render(){
  return(
    <div>
      <ChildComponent hChange={this.handleChange} />
    
</div>
  )
}

class ChildComponent extends React.Component{
  constructor(props){
    super(props);
     this.setState={
       input: ‘Return’’
    }

    this.cbackFunc = this.cbackFunc.bind(this);
  }
cbackFunc(){
  this.props.hChange(this.state.input);
}
render(){
  return(
    <div onClick={this.cbackFunc} />Click me</div>
  )
}
}
