Overriding Default Props

by Steve Golden

11/2/18 at 1:00p.m

To override default Props you will just pass in a value to a prop through the component. If you have multiple default props you can still use some of the default props. Here is an example:

Class Component extends React.Component{
  constructor(){
    super(props);
  }
  render(){
    return(
      <div>
        {/* Only passing the name prop not the mcount prop */}
        <MyComponent name=“Jason Jones” />
      </div>
    );
  }
}

const MyComponent = (props) => {
    {/* The name prop value will be what we passed through the component and the mcount value will be the default */}
  <p>Welcome back {props.name}, you have {props.mcount} messages.</p>
}

MyComponent.defaultProps = {
  name: ‘guest’,
  mcount: 0
}

Output will be:

Welcome back Jason Jones, you have 0 messages.

If you have to pass information through multiple components you will have to use both state and props methods. Usually the top/parent component will be your state since you will want your state information to be able to access the rest of your components. In addition, you rather have your child components to likely be more props because the state should be in parent components.

When passing the information you will use state syntax in the Parent Component and props syntax with the Child Component. Once the information is passed to the Child Component it is no longer state but a prop. Below is an example of how it will work:

Class ParentComponent extends React.Component{
  constructor(props){

      name='Tom Brady'
    }
  }
  render(){
    return(
       <div>
         <ChildComponent qname={this.state.name} />
       </div>
    );
  }
}

Function ChildComponent(props){
    return(
{/* Because we gave qname as the prop name in ParentComponent you have to use it instead of the state name*/}
       <p>My favorite quarterback is {this.props.qname}</p>
     );
}
